# TpCode 原型图表字段说明

## 1. 文档目的

本文档说明当前“原型图上传 / 展示 / 删除”功能涉及的核心数据表 `tp_prototype_asset` 的字段含义、当前使用情况和预留扩展方向，便于后续继续开发、联调和排查问题。

---

## 2. 核心主表

当前原型图功能的业务主表为：

- `tp_prototype_asset`

表定义位置：

- `packages/opencode/src/prototype/prototype.sql.ts`

该表用于记录：

- 原型图归属到哪个会话
- 原型图对应哪个页面
- 原型图版本号
- 原型图的本地存储路径
- 原型图展示地址
- 原型图来源类型
- 原型图创建与更新时间

---

## 3. 字段说明

### 3.1 主键与归属字段

#### `id`

- 含义：原型记录主键 ID
- 作用：唯一标识一张原型图记录
- 当前用途：查看原图、删除图片、后端读取文件

#### `session_id`

- 含义：所属会话 ID
- 作用：将原型图绑定到具体 session
- 当前用途：按会话查询原型图列表

#### `message_id`

- 含义：关联消息 ID
- 作用：预留给后续将原型图绑定到某次消息或某次任务输出
- 当前用途：当前版本未在前端实际使用

#### `user_id`

- 含义：操作人 ID
- 作用：记录是谁上传或生成了该原型图

#### `org_id`

- 含义：所属组织 ID
- 作用：记录原型图归属组织

#### `department_id`

- 含义：所属部门 ID
- 作用：记录原型图归属部门

---

### 3.2 页面识别字段

#### `agent_mode`

- 含义：生成时的模式
- 作用：标记当前保存发生在什么模式下
- 当前用途：主要用于后端模式校验

#### `title`

- 含义：原型标题
- 作用：给用户看，显示这张原型图叫什么
- 当前用途：前端卡片标题展示

#### `description`

- 含义：原型说明
- 作用：补充说明原型图内容
- 当前用途：预留字段，当前前端未重点使用

#### `route`

- 含义：页面路由
- 作用：记录该原型图对应哪个业务页面或菜单路径
- 当前用途：前端展示“页面路由”

#### `page_key`

- 含义：页面编码
- 作用：用于同一页面的版本归组
- 当前用途：
  - 前端展示“页面编码”
  - 后端按 `session_id + page_key` 递增版本号

---

### 3.3 画面采集信息字段

#### `viewport_width`

- 含义：视口宽度
- 作用：记录截图或上传时的页面宽度信息

#### `viewport_height`

- 含义：视口高度
- 作用：记录截图或上传时的页面高度信息

#### `device_scale_factor`

- 含义：设备缩放比
- 作用：记录截图时的缩放比例

---

### 3.4 文件与存储字段

#### `mime`

- 含义：图片 MIME 类型
- 作用：记录图片格式，例如 `image/png`

#### `size_bytes`

- 含义：图片字节大小
- 作用：记录图片文件大小

#### `storage_driver`

- 含义：存储驱动
- 作用：标记当前文件使用什么方式存储
- 当前实现：`local`

#### `storage_key`

- 含义：实际存储路径键
- 作用：记录原型图落盘位置
- 当前用途：
  - 后端读取文件
  - 前端展示“保存路径”

#### `image_url`

- 含义：原图访问地址
- 作用：访问原始图片文件
- 当前用途：前端“查看原图”

#### `thumbnail_url`

- 含义：缩略图访问地址
- 作用：列表预览图展示
- 当前用途：前端卡片预览

---

### 3.5 来源与测试字段

#### `source_type`

- 含义：来源类型
- 可选值：
  - `manual_upload`
  - `playwright_capture`
- 当前用途：前端展示“上传图片”或“截图保存”

#### `source_url`

- 含义：来源地址
- 作用：记录截图模式下的页面 URL
- 当前用途：后端保存，前端当前未展示

#### `test_run_id`

- 含义：测试运行 ID
- 作用：预留给后续“改完并测试后保存原型图”链路
- 当前用途：预留字段

#### `test_result`

- 含义：测试结果
- 可选值：
  - `passed`
  - `failed`
  - `unknown`
- 当前用途：预留字段

---

### 3.6 版本与状态字段

#### `version`

- 含义：版本号
- 作用：同一页面编码下的原型图自动递增版本
- 当前用途：前端展示 `v1 / v2 / v3`

#### `is_latest`

- 含义：是否最新版本
- 作用：标记当前页面编码下最新一张原型图
- 当前用途：前端展示“最新版本”

#### `status`

- 含义：记录状态
- 可选值：
  - `ready`
  - `archived`
  - `deleted`
- 当前用途：删除后标记状态，不直接物理删除表记录

---

### 3.7 时间字段

#### `time_created`

- 含义：创建时间
- 当前用途：前端展示上传时间

#### `time_updated`

- 含义：更新时间
- 当前用途：版本切换、删除、状态更新时刷新

---

## 4. 当前前端实际使用字段

当前 Web 原型图面板实际使用的字段主要有：

- `id`
- `title`
- `route`
- `page_key`
- `version`
- `is_latest`
- `storage_key`
- `image_url`
- `thumbnail_url`
- `source_type`
- `time_created`

这些字段对应当前页面上的：

- 标题展示
- 页面路由展示
- 页面编码展示
- 版本号展示
- 最新版本标记
- 保存路径展示
- 原图查看
- 缩略图展示
- 来源展示
- 时间展示

---

## 5. 当前已入库但前端未重点使用字段

以下字段当前已经保存到数据库，但前端暂未重点展示：

- `description`
- `message_id`
- `agent_mode`
- `viewport_width`
- `viewport_height`
- `device_scale_factor`
- `source_url`
- `test_run_id`
- `test_result`

这些字段主要用于：

- 后续业务扩展
- 后端追踪
- 调试排查
- 自动化联动能力预留

---

## 6. 当前预留字段

如果按现阶段功能划分，下面字段属于明显预留：

- `message_id`
  后续可绑定到某次对话消息

- `description`
  后续可承载页面说明、设计说明

- `test_run_id`
  后续可接测试任务记录

- `test_result`
  后续可接测试通过/失败结果

- `source_url`
  后续可在页面展示“截图来源页面”

- `agent_mode`
  后续可用于统计 build / plan 来源差异

---

## 7. 当前功能最核心字段

如果仅从“上传、展示、删除、版本管理”四个角度看，当前最核心字段为：

- `id`
- `session_id`
- `title`
- `route`
- `page_key`
- `storage_key`
- `image_url`
- `thumbnail_url`
- `version`
- `is_latest`

这组字段已经足够支撑当前功能闭环。

---

## 8. 原型图功能关联表

除主表 `tp_prototype_asset` 外，这条功能链还会读到或关联到以下表：

- `session`
  原型图归属会话

- `tp_audit_log`
  上传、截图、删除会写审计日志

- `tp_user`
  记录操作人

- `tp_organization`
  记录所属组织

- `tp_department`
  记录所属部门

说明：

- 图片文件本体不存数据库，而是落地到本地文件目录 `.opencode/prototypes/...`
- `message_id` 当前只是记录字段，不是强外键绑定到消息表

---

## 9. 总结

当前原型图功能在数据库层面，真正的业务核心主表只有一张：

- `tp_prototype_asset`

这张表已经覆盖：

- 会话归属
- 页面归属
- 文件存储
- 版本管理
- 来源追踪
- 删除状态
- 时间记录

现阶段结构已经可以满足：

- `build` 模式上传原型图
- `build` 模式删除原型图
- `plan/build` 模式查看原型图
- 同页面多版本管理

后续如果继续扩展“测试联动”“任务联动”“消息联动”，优先复用的也是这张表。
